详解hexo，快速上手，从使用到部署。

第一步：准备基础工具

  • 安装Git：Download git
  • 安装Node.js：Download nodejs
  • 安装Hexo：使用hexo

      Hexo可以方便快捷的搭建起一个静态网站，这三个工具的安装已经非常成熟，如果遇到问题可以通过Google、百度解决

第二部：Hexo的基本使用

  安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

第三步：新建完成后，指定文件夹的目录如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

_config.yml：
网站的 配置 信息，您可以在此配置大部分的参数。

package.json：
应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。

scaffolds：
模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source：
资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes：
主题 文件夹。Hexo 会根据主题来生成静态页面。

通过以上步骤就已经生成了静态网站，通过命令 hexo server 可以通过访问本地的链接进行查看。接下来可以通过 https://hexo.io/zh-cn/docs/commands 查看每个文件的具体信息。

更换主题的时候有要注意一下：Hexo的主题模板可以到github搜索。例如：https://github.com/litten/hexo-theme-yilia 这种的路径

第五步: 为主题添加自己想要的按钮。

更换了主题，发现新的主题没有归档、分类、标签、关于。想要添加这些可以通过 如下方法：

1：添加 关于页面
使用：hexo new page “about” 新建一个 关于页面。
主题的 _config.yml 文件中的 menu 中进行匹配

2：添加 标签页面
使用： hexo new page tags 新建一个 标签 页面。
主题的 _config.yml 文件中的 menu 中进行匹配

3：添加 分类页面
使用： hexo new page categories 新建一个 分类 页面。
主题的 _config.yml 文件中的 menu 中进行匹配

4：添加 归档页面
主题的 _config.yml 文件中的 menu 中进行匹配

5：添加 自定义页面
使用： hexo new page “guestbook” 新建一个 自定义 页面。
主题的 _config.yml 文件中的 menu 中进行匹配
